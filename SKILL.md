# PM Distill · 工作经验提炼技能

## 概述

帮助 B端 PM（供应链领域）把每次工作对话中的隐性判断，提炼成可携带、可复用的经验条目。
核心约束：**口语化 + 字数限制**，目的是让用户能默读记忆，不依赖笔记。

---

## 使用流程

### Step 1 · 判断场景，选择模板

根据用户描述的工作内容，选择对应模板：

| 场景 | 使用模板 |
|------|----------|
| 需求分析、PRD、业务判断、方案设计 | **模板 A · 标准提炼** |
| 用 AI / Claude Code 完成某个任务 | **模板 B · AI经验提炼** |
| 用户很忙，只想快速记一句话 | **模板 C · 最小提炼** |

如果用户没有明确说明场景，先问一句："这次是什么类型的工作？"，根据回答选模板。

---

### Step 2 · 执行对应模板

#### 模板 A · 标准经验提炼（日常使用）

输出以下四项，**总字数200字以内，口语化**：

```
【核心问题】一句话描述这次任务/问题的本质

【判断逻辑】
1. ...
2. ...
3. ...（最多3步）

【可复用框架】如果有，用一句话描述；没有则写"本次无明显框架"

【下次注意】下次遇到类似情况，最重要的一个注意点
```

#### 模板 B · AI应用经验提炼

输出以下四项，**总字数150字以内，重点在可复用性**：

```
【AI价值】这个任务用AI做的核心价值（vs 不用AI直接做）

【Prompt关键】什么样的提问方式起了关键作用

【局限/踩坑】遇到的问题或局限（没有则写"本次无明显坑"）

【可推广场景】这个方法还可以用在哪些类似场景
```

#### 模板 C · 最小提炼（30秒版）

只输出一句话，**20字以内**：

```
【今日最值得记住的判断】___
```

---

### Step 3 · 输出后追加「记忆钩子」

每次提炼完成后，在输出末尾加一行：

```
💡 记忆钩子：[用5个字以内概括这条经验的关键词，方便用户在脑子里快速检索]
```

例：`💡 记忆钩子：先确认边界`、`💡 记忆钩子：AI适合归纳`

---

## 输出原则

- **口语化优先**：避免书面腔，用说话的方式写
- **字数是硬约束**：超出字数上限时，主动删减，不保留冗余
- **不编造内容**：如果用户描述不够清晰，只提炼已知信息，对不确定的部分标注「待补充」
- **不加废话**：不要在输出前说"好的，我来帮你提炼"，直接给结果

---

## 快速触发示例

用户说以下任何一种，立即用对应模板响应：

- "帮我提炼一下" → 问清楚场景后用模板A或B
- "今天的复盘" → 模板A
- "这次AI用的效果不错，记录一下" → 模板B  
- "没时间，给我一句话" → 模板C
- "distill" → 模板A（默认）
- "今天就记这一条" → 模板C
